कोई प्रॉम्प्ट पेस्ट करें जो आपने पहले से लिखा है। प्रॉम्प्ट डॉक्टर इसे Claude के लिए मायने रखने वाली प्रॉम्प्टिंग प्रथाओं के विरुद्ध स्कोर करता है, ठीक-ठीक बताता है कि क्या कमज़ोर है, हर निष्कर्ष को उस पेज से जोड़ता है जो उसे ठीक करता है, और कॉपी करने के लिए आपको एक बेहतर स्कैफोल्ड देता है।

यह **100% आपके ब्राउज़र में** चलता है — आपका प्रॉम्प्ट कहीं भी नहीं भेजा जाता। ये जाँचें सादे ह्यूरिस्टिक्स हैं, कोई AI कॉल नहीं: तेज़, मुफ़्त, और पारदर्शी।

<PromptDoctor />

## यह क्या जाँचता है

प्रत्येक नियम इस साइट पर एक गहरे पेज से मैप होता है:

| जाँच | यह क्यों मायने रखता है |
| --- | --- |
| **कार्रवाई के लिए पर्याप्त** | एक-पंक्ति वाले प्रॉम्प्ट Claude को आपके इरादे का अनुमान लगाने पर मजबूर करते हैं। |
| **एक भूमिका तय करता है** | "आप एक वरिष्ठ X हैं" टोन और निर्णय को तेज़ी से केंद्रित करता है — देखें [भूमिकाएँ](/docs/foundations/roles)। |
| **एक स्पष्ट कार्य बताता है** | एक अग्रणी आदेशात्मक क्रिया ("सारांश दें…", "निकालें…") अस्पष्टता को हटा देती है। |
| **आउटपुट का स्वरूप निर्दिष्ट करता है** | किसी प्रारूप के बिना आपको वही लंबाई और संरचना मिलती है जो Claude चुनता है — देखें [संरचित आउटपुट](/docs/api/structured-output)। |
| **एक उदाहरण दिखाता है** | पैटर्न कार्यों के लिए, एक इनपुट→आउटपुट जोड़ी विवरण के पैराग्राफों से बेहतर है — देखें [फ़्यू-शॉट](/docs/prompting/few-shot)। |
| **लंबे होने पर संरचित** | लंबे प्रॉम्प्ट को [XML टैग](/docs/prompting/xml-tags) की ज़रूरत होती है ताकि Claude निर्देशों को सामग्री से अलग कर सके। |
| **मनगढ़ंत उत्तरों से बचाव** | जब आप स्रोत टेक्स्ट देते हैं, तो Claude को केवल उसी का उपयोग करने को कहें — यह [भ्रम](/docs/foundations/hallucinations) के विरुद्ध मुख्य बचाव है। |
| **अस्पष्ट विशेषणों से बचता है** | "अच्छा", "आकर्षक", "पेशेवर" मापने योग्य नहीं हैं। इन्हें परखने योग्य मानदंडों से बदलें। |

:::tip स्कोर से शिप तक
उच्च स्कोर लक्ष्य नहीं है — एक भरोसेमंद परिणाम है। स्कैफोल्ड को एक शुरुआती बिंदु मानें, फिर आउटपुट पर निर्भर होने से पहले इसे एक असली [eval](/docs/foundations/evals) से सिद्ध करें।
:::
