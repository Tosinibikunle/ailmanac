अनुमतियाँ यह तय करती हैं कि **Claude Code आपसे पूछने के लिए रुके बिना क्या कर सकता है**। इन्हें अच्छी तरह ट्यून करें तो आपको नियंत्रण खोए बिना प्रवाह मिलता है; इन्हें बुरी तरह ट्यून करें तो या तो आप हर चीज़ पर रबर-स्टैम्प लगाते हैं या प्रॉम्प्ट्स में डूब जाते हैं।

## अनुमति मोड

एक मोड एक सत्र के लिए समग्र स्थिति निर्धारित करता है:

| मोड | व्यवहार | कब उपयोग करें |
|---|---|---|
| **default / ask** | किसी भी पूर्व-अनुमत न की गई चीज़ के लिए प्रॉम्प्ट करता है | रोज़मर्रा का काम |
| **plan** | केवल-पठन; प्रस्ताव देता है, कभी कार्य नहीं करता | बड़े/जोखिम भरे कार्य — देखें [Plan Mode](/docs/claude-code/plan-mode) |
| **acceptEdits** | फ़ाइल संपादनों को स्वतः स्वीकार करता है | एक विश्वसनीय, अच्छी तरह सीमित संपादन सत्र |
| **bypass / yolo** | प्रॉम्प्ट्स को पूरी तरह छोड़ देता है | केवल सैंडबॉक्स/CI — सीक्रेट्स वाली मशीन पर **कभी नहीं** |

<Callout type="warning" items={["Bypass का स्थान एक सैंडबॉक्स में है। अपनी वास्तविक मशीन पर सभी प्रॉम्प्ट बंद करके चलाना ही वह तरीका है जिससे एक एजेंट किसी ऐसी चीज़ को छू बैठता है जिसे नहीं छूना चाहिए। इसे डिस्पोज़ेबल वातावरण के लिए सुरक्षित रखें। /docs/security/hardening-autonomous-runs पर स्वायत्त रन को मज़बूत बनाना देखें।"]} />

## एक समझदार शुरुआती allowlist

लक्ष्य: सुरक्षित, दोहराव वाली चीज़ों को पूर्व-अनुमति दें; विनाशकारी चीज़ों को **ask** या **deny** पर रखें।

<Steps items={[{title: "सुरक्षित + दोहराव वाली चीज़ों को allow करें", body: "फ़ाइलें पढ़ें, अपने test/lint/build कमांड चलाएँ, git status/diff।"}, {title: "मध्यम-जोखिम वाली चीज़ों से पहले पूछें", body: "डिपेंडेंसी इंस्टॉल करना, प्रोजेक्ट के बाहर फ़ाइलें लिखना, नेटवर्क कॉल।"}, {title: "विनाशकारी चीज़ों को deny करें", body: "सीक्रेट फ़ाइलें पढ़ना (.env, key फ़ाइलें), force-push, rm -rf।"}]} />

प्रोजेक्ट नियमों को [`settings.json`](/docs/claude-code/settings) (साझा) में और व्यक्तिगत ओवरराइड्स को `settings.local.json` में संग्रहीत करें।
