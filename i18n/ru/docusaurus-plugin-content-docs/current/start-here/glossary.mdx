### Агент (Agent)
ИИ, настроенный так, чтобы **выполнять действия в цикле**: он читает цель, вызывает инструменты, видит результаты и повторяет до завершения. Claude Code — это агент для вашей кодовой базы.

### Anthropic
Компания, которая создаёт Claude.

### API (программный интерфейс приложения)
Способ для *вашего* программного обеспечения общаться с Claude программно, а не через ввод в окне чата. См. [Ваш первый вызов API](/docs/api/first-call).

### Артефакт (Artifact)
Живой, исполняемый результат (приложение, диаграмма или документ), который Claude отображает в боковой панели на claude.ai отдельно от чата.

### Цепочка рассуждений (Chain-of-thought, CoT)
Просьба к модели **рассуждать шаг за шагом** перед ответом, что повышает точность на сложных задачах.

### Claude
Семейство ИИ-моделей (и приложений), созданных Anthropic. «Claude» может означать чат-приложение, модели или бренд в зависимости от контекста.

### Claude Code
**Агентный инструмент командной строки/IDE** от Anthropic, который может читать и редактировать вашу кодовую базу, выполнять команды и использовать инструменты. См. [Что такое Claude Code](/docs/claude-code/what-is-claude-code).

### CLAUDE.md
Текстовый файл, в котором вы даёте Claude Code постоянные инструкции о вашем проекте — это его настройка с наибольшей отдачей.

### Уплотнение (Compaction)
Автоматическое сжатие более старых реплик разговора в краткое резюме, когда [контекстное окно](/docs/foundations/tokens-and-context) заполняется, чтобы длинная сессия могла продолжаться, не теряя нить. См. [Управление контекстом](/docs/claude-code/context-management).

### Инженерия контекста (Context engineering)
Осознанное решение о том, что попадает в [контекстное окно](/docs/foundations/tokens-and-context) модели — инструкции, примеры, извлечённые данные — чтобы получить наилучший результат. Дисциплина, стоящая за промптингом, RAG и памятью.

### Контекстное окно (Context window)
Максимальный объём текста (измеряемый в **токенах**), который модель может учитывать одновременно — ваш запрос *плюс* её ответ. Превысите его — и самое старое содержимое отбрасывается.

### Усилие (Effort)
Регулятор в более новых моделях Claude, задающий примерно **то, насколько модель размышляет** перед ответом. Подбирайте уровень под задачу, а не выкручивайте его на максимум — за размышление вы платите задержкой и стоимостью. См. [Расширенное мышление и усилие](/docs/api/thinking-and-effort).

### Эмбеддинг (Embedding)
Список чисел, представляющий *смысл* фрагмента текста, чтобы программное обеспечение могло находить вещи, близкие по смыслу (а не просто совпадающие по ключевым словам). Лежит в основе семантического поиска и RAG.

### Eval (оценка)
Воспроизводимый тест качества вывода ИИ, чтобы вы могли понять, стало ли лучше или хуже после изменения.

### Few-shot
Включение **нескольких готовых примеров** в ваш запрос, чтобы показать модели нужный вам шаблон.

### Дообучение (Fine-tuning)
Дополнительное обучение модели на ваших собственных примерах для её специализации. Обычно это крайняя мера — сначала [попробуйте промптинг и RAG](/docs/foundations).

### Заземление (Grounding)
Указание модели отвечать **только на основе предоставленного вами контекста** — и говорить «я не знаю», когда ответа там нет. Главная защита на уровне промпта против галлюцинаций. См. [Библиотеку паттернов](/docs/prompting/pattern-library).

### Ограждения (Guardrails)
Правила и ограничения, которые удерживают поведение ИИ безопасным и в рамках задачи — например, правила разрешений, «запусти тесты перед завершением» или одобрение человеком для рискованных действий. См. [Защита агентов](/docs/security/securing-agents).

### Галлюцинация (Hallucination)
Когда модель утверждает что-то **ложное, но уверенно**. Ключевой навык — проверка. См. [Галлюцинации](/docs/foundations).

### Хук (Hook)
В Claude Code — команда, которая выполняется **автоматически** при событии жизненного цикла (например, после каждого редактирования файла), чтобы обеспечить форматирование, проверки или безопасность.

### Инференс (Inference)
Запуск модели для получения вывода (в отличие от её обучения).

### Задержка (Latency)
Сколько вы ждёте ответа. Более крупные модели обычно медленнее; [потоковая передача](/docs/api/streaming) улучшает *воспринимаемую* задержку, показывая вывод по мере его генерации.

### LLM (большая языковая модель)
Тип ИИ, стоящий за Claude: обучен **предсказывать следующий фрагмент текста**, чего оказывается достаточно, чтобы писать, программировать, резюмировать и рассуждать. См. [Что такое LLM?](/docs/foundations).

### MCP (Model Context Protocol)
**Открытый стандарт** для подключения ИИ к внешним инструментам и данным (базам данных, GitHub, браузерам). «MCP-сервер» предоставляет инструмент; Claude может его вызвать. См. [MCP](/docs/claude-code).

### Мультимодальность (Multimodal)
Модель, которая работает **не только с текстом** — например, принимает изображения и PDF на вход.

### Стиль вывода (Output style)
В Claude Code — пресет, который меняет **то, как Claude общается** (например, кратко, с пояснениями, в обучающем режиме), не меняя того, что он умеет делать. См. [Стили вывода](/docs/claude-code/output-styles).

### Режим разрешений (Permission mode)
В Claude Code — то, насколько много он может делать **без спроса** — от запроса на каждое действие до автоматического принятия правок. Главный регулятор баланса между скоростью и безопасностью. См. [Разрешения](/docs/claude-code/permissions).

### Режим планирования (Plan mode)
Режим Claude Code только для чтения, который исследует и предлагает план **прежде**, чем коснётся каких-либо файлов — вы одобряете план до того, как он начнёт действовать. См. [Режим планирования](/docs/claude-code/plan-mode).

### Промпт (Prompt)
То, что вы отправляете модели: ваши инструкции, вопрос и любой контекст.

### Кэширование промптов (Prompt caching)
Повторное использование неизменной начальной части промпта между вызовами для снижения стоимости и задержки. См. [Кэширование промптов](/docs/api).

### Инъекция промпта (Prompt injection)
Атака, при которой **вредоносные инструкции, спрятанные в контенте**, который читает ИИ (веб-страница, документ), пытаются перехватить управление им. См. [Инъекция промпта](/docs/security).

### RAG (генерация с дополненной выборкой)
Извлечение релевантных фрагментов из *ваших* данных и добавление их в промпт, чтобы модель отвечала именно на их основе. См. [RAG](/docs/foundations).

### Рассуждение / расширенное мышление (Reasoning / extended thinking)
Режим, в котором модель тратит дополнительные вычисления на «обдумывание» перед ответом, для более сложных задач. В более новых моделях вы настраиваете его с помощью [усилия](/docs/api/thinking-and-effort).

### Навык (Skill)
Упакованная возможность (инструкции + опциональные скрипты), которую Claude загружает **по требованию**, когда это уместно. См. [Навыки](/docs/claude-code).

### Слэш-команда (Slash command)
Ярлык с префиксом `/` в Claude Code — встроенный (например, `/clear`) или ваш собственный многоразовый промпт — который запускает заданное действие. См. [Слэш-команды](/docs/claude-code/slash-commands).

### Структурированный вывод (Structured output)
Получение результатов в строгом, машиночитаемом виде (например, JSON, соответствующий схеме) вместо свободного текста — чтобы на них могло полагаться другое программное обеспечение. См. [Структурированный вывод](/docs/api/structured-output).

### Субагент (Subagent)
Отдельный экземпляр Claude со своим контекстом и инструментами, используемый для **делегирования** части задачи (например, исследования), чтобы основная сессия оставалась сфокусированной.

### Системный промпт (System prompt)
Высокоуровневые инструкции, задающие роль модели и правила для всего разговора — ваш самый мощный рычаг управления.

### Температура (Temperature)
Параметр (у некоторых моделей), управляющий случайностью: низкая = сфокусированно и воспроизводимо, высокая = разнообразно и креативно.

### Токен (Token)
Единица, которую модели читают и считают, — примерно ¾ слова в английском языке. Цены и лимиты контекста измеряются в токенах. См. [Токены и контекст](/docs/foundations).

### Использование инструментов / вызов функций (Tool use / function calling)
Возможность для модели вызывать определённые вами функции (поиск, калькулятор, ваш API) и использовать результаты. См. [Использование инструментов](/docs/api).
