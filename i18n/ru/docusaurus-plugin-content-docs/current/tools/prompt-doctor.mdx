Вставьте промпт, который вы уже написали. Prompt Doctor оценивает его по практикам промптинга, важным для Claude, точно отмечает слабые места, связывает каждую находку со страницей, которая её исправляет, и выдаёт улучшенный каркас для копирования.

Он работает **на 100% в вашем браузере** — ваш промпт никуда не отправляется. Проверки — это обычные эвристики, а не вызов ИИ: быстро, бесплатно и прозрачно.

<PromptDoctor />

## Что он проверяет

Каждое правило ведёт к более подробной странице на этом сайте:

| Проверка | Почему это важно |
| --- | --- |
| **Достаточно для действия** | Однострочные промпты заставляют Claude угадывать ваше намерение. |
| **Задаёт роль** | «Ты старший специалист по X» резко фокусирует тон и суждения — см. [Роли](/docs/foundations/roles). |
| **Формулирует одну ясную задачу** | Ведущий повелительный глагол («Суммируй…», «Извлеки…») снимает неоднозначность. |
| **Указывает форму вывода** | Без формата вы получаете ту длину и структуру, которые выберет Claude — см. [Структурированный вывод](/docs/api/structured-output). |
| **Показывает пример** | Для шаблонных задач одна пара вход→выход лучше абзацев описания — см. [Few-Shot](/docs/prompting/few-shot). |
| **Структурирован, когда длинный** | Длинным промптам нужны [XML-теги](/docs/prompting/xml-tags), чтобы Claude мог отличить инструкции от содержимого. |
| **Защищает от выдуманных ответов** | Когда вы даёте исходный текст, скажите Claude использовать только его — основная защита от [галлюцинаций](/docs/foundations/hallucinations). |
| **Избегает расплывчатых уточнений** | «хороший», «увлекательный», «профессиональный» неизмеримы. Замените их проверяемыми критериями. |

:::tip От оценки к запуску
Высокая оценка — не цель, цель — надёжный результат. Относитесь к каркасу как к отправной точке, затем подтвердите его реальным [eval](/docs/foundations/evals), прежде чем полагаться на вывод.
:::
