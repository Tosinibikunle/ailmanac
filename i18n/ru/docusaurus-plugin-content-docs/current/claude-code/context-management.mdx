## Главная идея

У каждой сессии есть конечное [контекстное окно](/docs/foundations/tokens-and-context). По мере его заполнения более старое содержимое вытесняется, и Claude может начать «дрейфовать». Осознанное управление контекстом — это то, что поддерживает остроту длинных сессий.

## Две основные команды

Две команды покрывают почти любую ситуацию. Хитрость в том, чтобы сопоставить команду с вашим намерением.

<Flashcards cards={[{front: "/clear", back: "Стереть разговор и начать заново. Используйте, когда переключаетесь на НЕ СВЯЗАННУЮ задачу. Самый дешёвый способ вернуть чистый, сфокусированный контекст."}, {front: "/compact", back: "Резюмировать разговор на текущий момент в компактную форму — сохраняет суть, освобождая место. Решайте с помощью одного вопроса: это новая задача или та же самая?

```mermaid
flowchart TD
  A{New task or same task?} -->|New / unrelated| B[/clear]

```

## Авто-уплотнение

Когда контекст приближается к пределу, Claude Code может **автоматически уплотнить** его, чтобы работа продолжалась без жёсткой остановки. Это страховочная сетка — но проактивное `/clear` между задачами даёт лучшие результаты, чем опора на неё.

<Callout type="tip" items={["Относитесь к авто-уплотнению как к подстраховке, а не как к стратегии. Очистка между несвязанными задачами поддерживает контекст чище, чем когда окно заполняется до срабатывания авто-уплотнения."]} />

:::tip Симптом → решение
Если Claude начинает забывать более раннюю инструкцию или повторяться, окно, скорее всего, заполнено. `/compact` (та же задача) или `/clear` (новая задача) обычно решают это мгновенно. Подробнее в [Устранении неполадок](/docs/contribute/troubleshooting).
:::
