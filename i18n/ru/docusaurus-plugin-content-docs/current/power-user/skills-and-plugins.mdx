Самый быстрый способ прокачать Claude Code — перестать писать всё вручную и **установить возможности, которые другие люди уже довели до совершенства**. Навыки дают Claude экспертизу по запросу; плагины объединяют навыки, агентов, команды, хуки и MCP-серверы в один версионируемый, устанавливаемый блок. Эта страница — кураторский, готовый к установке шорт-лист: что делает каждая вещь, когда к ней обращаться и точная команда для добавления.

Если механика для вас всё ещё расплывчата, сначала прочитайте [Навыки](/docs/claude-code/skills) и [Плагины и маркетплейсы](/docs/claude-code/plugins-marketplaces) — эта страница предполагает, что вы знаете, что такое `SKILL.md`.

<VerifyNote lastVerified="2026-06-24" source="https://code.claude.com/docs/en/discover-plugins">
Названия маркетплейсов, каталог официальных плагинов и количество звёзд у репозиториев сообщества часто меняются. Плагины выполняют код с вашими привилегиями — устанавливайте только из источников, которым доверяете, и подтверждайте актуальные команды в официальной документации, прежде чем на них полагаться.
</VerifyNote>

## Навык против плагина: модель на 20 секунд

**Навык** — это папка с `SKILL.md` (фронтматтер `description` + инструкции). Claude загружает его только когда он релевантен, либо вы вызываете его через `/skill-name`. Он находится в `~/.claude/skills/` (личный), в `.claude/skills/` проекта или внутри плагина. Согласно официальной документации, пользовательские слэш-команды были объединены с навыками — и `.claude/commands/deploy.md`, и `.claude/skills/deploy/SKILL.md` создают `/deploy`.

**Плагин** — это распространяемый набор, который может поставлять навыки, агентов, хуки, MCP-серверы и LSP-серверы вместе, с манифестом `.claude-plugin/plugin.json`. Плагины вы получаете из **маркетплейса** — каталога, который добавляете один раз, а затем устанавливаете из него. Навыки плагинов имеют пространство имён, например `/commit-commands:commit`.

## Официальные плагины, которые стоит установить первыми

Эти плагины поставляются в официальном маркетплейсе Anthropic (`claude-plugins-official`) и являются наиболее ценной отправной точкой. Устанавливайте через `/plugin install <name>@claude-plugins-official`.

### Интеллект кода (LSP) — пусть Claude мгновенно видит ошибки типов

Плагины LSP подключают языковой сервер к Claude Code, так что после каждого редактирования Claude получает автоматическую диагностику (ошибки типов, отсутствующие импорты) и точную навигацию (переход к определению, поиск ссылок) вместо догадок через grep. Им нужен соответствующий бинарник языкового сервера на вашей машине.

- **Когда использовать:** любая нетривиальная сессия кодирования на типизированном языке. Это, вероятно, самое большое отдельное повышение точности для работы над разработкой.
- **Компромисс:** серверы вроде `rust-analyzer` и `pyright` могут потреблять много памяти на больших репозиториях; отключайте, если это мешает.

<PromptCard title="Установите LSP для вашего стека">
{`/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls`}
</PromptCard>

### Процесс работы с Git и PR

- **`commit-commands`** — git-процессы: индексация, генерация сообщения коммита, push, открытие PR. Используйте, когда хотите чистые, единообразные коммиты без набора церемоний. Навыки имеют пространство имён, например `/commit-commands:commit`.
- **`pr-review-toolkit`** — специализированные агенты для ревью pull-request'ов. Используйте перед слиянием или чтобы получить взгляд со стороны на диф.

<PromptCard title="Основа для Git/PR">
{`/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins`}
</PromptCard>

### Автоматическое ревью безопасности

- **`security-guidance`** — проверяет каждое изменение, которое делает Claude, на распространённые уязвимости и инструктирует Claude исправить их в той же сессии. Используйте как постоянную защитную сеть в любом проекте с написанием кода; это дополняет (но не заменяет) настоящее ревью.

<PromptCard title="Постоянная проверка безопасности">
{`/plugin install security-guidance@claude-plugins-official`}
</PromptCard>

### Внешние интеграции (MCP, преднастроенные)

Эти плагины включают готовый к работе [MCP-сервер](/docs/claude-code/mcp), так что вы пропускаете ручную настройку. Обращайтесь к одному из них, когда хотите, чтобы Claude действовал прямо в этом сервисе.

- **Контроль версий:** `github`, `gitlab`
- **Управление проектами:** `linear`, `notion`, `asana`, `atlassian` (Jira/Confluence)
- **Инфраструктура:** `vercel`, `firebase`, `supabase`
- **Дизайн / коммуникации / мониторинг:** `figma`, `slack`, `sentry`

<PromptCard title="Подключите Claude к вашим инструментам">
{`/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official`}
</PromptCard>

### Наборы инструментов для создания собственных решений и стили вывода

- **`plugin-dev`** — набор инструментов для создания собственных плагинов. Используйте в тот момент, когда вашей настройкой `.claude/` стоит поделиться.
- **`agent-sdk-dev`** — инструменты для разработки на основе Claude Agent SDK.
- **`explanatory-output-style` / `learning-output-style`** — меняют *то, как* Claude отвечает: пояснения по ходу дела или интерактивный режим обучения. Отлично подходит для онбординга или когда вы хотите действительно понять изменения.

## Ценные навыки по принципу «задача, которую нужно решить»

Навыки блистают на повторяемых задачах с насыщенным контекстом. Anthropic публикует публичный справочный набор в репозитории [`anthropics/skills`](https://github.com/anthropics/skills) — те же паттерны, что обеспечивают документные и творческие возможности Claude. Вы можете скопировать папку навыка в `~/.claude/skills/` (личный) или `.claude/skills/` (проект), и она загрузится в следующей сессии.

| Задача | Навык (из anthropics/skills) | Когда к нему обращаться |
| :-- | :-- | :-- |
| **Документы** | `docx`, `pdf`, `pptx`, `xlsx` | Генерировать или редактировать настоящие файлы Office/PDF — отчёты, презентации, таблицы, заполняемые формы. (Эталонные реализации с открытым исходным кодом.) |
| **Создание интерфейсов** | `web-artifacts-builder` / `artifacts-builder` | Создавать самодостаточные HTML/JS-артефакты и мини-приложения. |
| **Расширение Claude** | `mcp-builder` | Создать каркас нового MCP-сервера, когда интеграции ещё не существует. |
| **Бренд и коммуникации** | навыки стиля brand-guidelines / internal-comms | Поддерживать тон, наименования и форматирование в духе бренда во всех результатах. |

<Callout type="tip" items={["Не сваливайте 30 навыков в ~/.claude/skills сразу. Навыки используют прогрессивное раскрытие — пока навык не сработал, загружаются только имя + описание — но раздутая куча с расплывчатыми описаниями делает срабатывание ненадёжным. Установите несколько, напишите чёткие описания «Use when…» и растите осознанно.", "AILmanac поставляет готовые наборы навыков, которые можно подключить и попробовать — см. /docs/templates/skills."]} />

### Самозапускающийся навык, который можно вставить прямо сейчас

Для работы с большим объёмом исследований навык, который заземляет Claude на свежих источниках, лучше, чем объяснять свой метод заново каждый раз. Поместите это в `~/.claude/skills/research-brief/SKILL.md`:

<PromptCard title="~/.claude/skills/research-brief/SKILL.md">
{`---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.`}
</PromptCard>

Существуют десятки репозиториев «awesome»; большинство из них — автоматически сгенерированные зеркала. Несколько по-настоящему кураторские и поддерживаемые — начните с них.

- **[hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code)** — самый известный собранный вручную указатель навыков, хуков, слэш-команд, оркестраторов агентов, приложений и плагинов. Десятки тысяч звёзд; активно реорганизуется.
- **[ComposioHQ/awesome-claude-skills](https://github.com/ComposioHQ/awesome-claude-skills)** — большой кураторский каталог навыков Claude, сгруппированных по функциям (документы, разработка, данные, маркетинг, безопасность, SaaS-автоматизация).
- **[VoltAgent/awesome-agent-skills](https://github.com/VoltAgent/awesome-agent-skills)** — 1400+ навыков сообщества и официальных (Anthropic, Vercel, Stripe, Cloudflare, Sentry, Figma и другие), переносимых между Claude Code, Cursor, Gemini CLI, Codex и прочими.
- **[anthropics/claude-plugins-community](https://github.com/anthropics/claude-plugins-community)** — официальный маркетплейс **плагинов** сообщества; сторонние плагины, прошедшие автоматическую проверку и отбор безопасности Anthropic.
