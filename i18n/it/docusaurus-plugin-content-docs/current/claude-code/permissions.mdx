## Modalità dei permessi

Una modalità imposta la postura complessiva per una sessione:

| Modalità | Comportamento | Usala quando |
|---|---|---|
| **default / ask** | Chiede conferma per tutto ciò che non è preautorizzato | Lavoro quotidiano |
| **plan** | Sola lettura; propone, non agisce mai | Task grandi/rischiosi — vedi [Modalità Plan](/docs/claude-code/plan-mode) |
| **acceptEdits** | Accetta automaticamente le modifiche ai file | Una sessione di editing fidata e ben circoscritta |
| **bypass / yolo** | Salta del tutto le richieste | Solo sandbox/CI — **mai** su una macchina con segreti |

## Una allowlist iniziale sensata

L'obiettivo: preautorizzare le cose sicure e ripetitive; tenere le cose distruttive su **ask** o **deny**.

<Steps items={[{title: "Consenti ciò che è sicuro + ripetitivo", body: "Leggere file, eseguire i tuoi comandi di test/lint/build, git status/diff."}, {title: "Chiedi prima di ciò che è a rischio medio", body: "Installare dipendenze, scrivere file fuori dal progetto, chiamate di rete."}, {title: "Nega ciò che è distruttivo", body: "Leggere file segreti (.env, file di chiavi), force-push, rm -rf."}]} />

Conserva le regole di progetto in [`settings.json`](/docs/claude-code/settings) (condivise) e gli override personali in `settings.local.json`.

<Callout type="tip" items={["Lascia che impari dalle tue richieste: approva lo stesso comando sicuro alcune volte e saprai esattamente cosa aggiungere alla tua allowlist — trasformando richieste ripetute in una regola una tantum."]} />
