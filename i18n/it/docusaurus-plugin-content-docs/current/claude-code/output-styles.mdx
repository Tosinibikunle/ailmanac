Gli **stili di output** cambiano il *modo* in cui Claude Code comunica — il suo ruolo, tono, prolissità e se spiega o meno il proprio ragionamento — modificando direttamente il system prompt. Cambiano il *modo* in cui Claude risponde, non *ciò* che sa. Ricorri a uno di essi quando ti ritrovi a richiedere ogni volta lo stesso registro o formato.

## Perché usarli

- **Adattali al tuo flusso di lavoro.** Conciso e orientato all'azione quando vai veloce; esplicativo quando stai imparando.
- **Modalità didattica.** Alcuni stili aggiungono commenti formativi — ottimi mentre prendi confidenza con una codebase o una tecnica.
- **Coerenza.** Imposta una volta lo stile che preferisci, così ogni sessione si presenta come piace a te.

## Gli stili integrati

Claude Code include quattro stili predefiniti:

| Stile | Cosa fa |
|---|---|
| **Default** | Il system prompt standard per l'ingegneria del software — efficiente, orientato all'azione. |
| **Proactive** | Esegue immediatamente e fa assunzioni ragionevoli invece di fermarsi per le decisioni di routine. Offre indicazioni di esecuzione autonoma più marcate rispetto alla modalità auto, ma vedi comunque le richieste di permesso prima che gli strumenti vengano eseguiti. |
| **Explanatory** | Aggiunge "Insights" formativi tra un passaggio e l'altro, spiegando le scelte implementative e i pattern della codebase. |
| **Learning** | Collaborativo, basato sull'imparare facendo: condivide Insights *e* ti chiede di scrivere tu stesso piccole parti, lasciando marcatori `[TODO(human)]` nel tuo codice. |

<Callout type="warning" items={[
  "Explanatory e Learning producono per loro natura risposte più lunghe, quindi costano più token di output rispetto a Default."
]} />

## Come passare da uno all'altro

Esegui `/config` e scegli uno stile sotto **Output style**. La tua scelta viene salvata in `.claude/settings.local.json` a livello di progetto. Per impostarlo senza il menu, modifica il campo `outputStyle` in qualsiasi file di impostazioni.

<Steps items={[
  { title: "Apri il menu di configurazione", body: "Esegui /config nella tua sessione e individua l'opzione Output style." },
  { title: "Scegli uno stile", body: "Seleziona uno tra Default, Proactive, Explanatory o Learning. La tua scelta viene salvata in .claude/settings.local.json a livello di progetto." },
  { title: "Oppure impostalo direttamente", body: "Salta il menu modificando il campo outputStyle in qualsiasi file di impostazioni (vedi il JSON qui sotto)." },
  { title: "Avvia una nuova sessione", body: "Lo stile fa parte del system prompt, letto una sola volta all'avvio della sessione — esegui /clear o apri una nuova sessione affinché abbia effetto." }
]} />

```json
{
  "outputStyle": "Explanatory"
}
```

:::warning Le modifiche richiedono una nuova sessione
Lo stile di output fa parte del system prompt, che Claude Code legge **una sola volta all'avvio della sessione**. Una modifica ha effetto solo dopo `/clear` o in una nuova sessione. (Il comando autonomo `/output-style` è stato rimosso nella v2.1.91 — usa `/config` o l'impostazione qui sopra.)
:::

## Creare uno stile personalizzato

Uno stile personalizzato è un file Markdown: frontmatter, poi le istruzioni da accodare al system prompt. Salvalo a livello utente (`~/.claude/output-styles/`) o a livello di progetto (`.claude/output-styles/`); il nome del file diventa il nome dello stile, a meno che tu non imposti `name`.

<PromptCard title="Esempio di stile personalizzato — Diagrammi prima di tutto">{`---

description: Lead every explanation with a diagram

---

When explaining code, architecture, or data flow, start with a Mermaid
diagram showing the structure, then explain in prose.`}</PromptCard>

## Stili di output vs CLAUDE.md vs altre leve

- **Stile di output** = un preset del system prompt per ruolo, tono e formato — si applica a *ogni* risposta.
- **[CLAUDE.md](/docs/claude-code/claude-md)** = fatti e regole *di progetto* persistenti (convenzioni, guardrail), aggiunti come messaggio dopo il system prompt.
- **[Istruzioni personalizzate / stili di risposta](/docs/claude-app/custom-instructions)** = il cugino dell'app consumer Claude.ai.

:::tip Usa la leva giusta
Vuoi che Claude "esegua sempre i test prima di concludere"? Quello è un **guardrail di CLAUDE.md**, non uno stile. Vuoi che "spieghi ogni passaggio man mano che procede"? Quello è uno **stile di output**. Tieni le regole di comportamento in CLAUDE.md e la presentazione negli stili.
:::
