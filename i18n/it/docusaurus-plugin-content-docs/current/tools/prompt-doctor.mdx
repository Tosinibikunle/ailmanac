Incolla un prompt che hai già scritto. Il Prompt Doctor lo valuta rispetto alle pratiche di prompting che contano per Claude, segnala con precisione i punti deboli, collega ogni rilievo alla pagina che lo risolve e ti consegna una struttura migliorata da copiare.

Funziona **al 100% nel tuo browser** — il tuo prompt non viene mai inviato da nessuna parte. I controlli sono semplici euristiche, non una chiamata a un'AI: veloci, gratuiti e trasparenti.

<PromptDoctor />

## Cosa controlla

Ogni regola rimanda a una pagina di approfondimento su questo sito:

| Controllo | Perché conta |
| --- | --- |
| **Abbastanza per agire** | I prompt di una sola riga costringono Claude a indovinare le tue intenzioni. |
| **Definisce un ruolo** | "Sei un senior X" focalizza nettamente tono e giudizio — vedi [Ruoli](/docs/foundations/roles). |
| **Indica un solo task chiaro** | Un verbo imperativo iniziale ("Riassumi…", "Estrai…") elimina l'ambiguità. |
| **Specifica la forma dell'output** | Senza un formato ottieni la lunghezza e la struttura che Claude decide a piacere — vedi [Output Strutturato](/docs/api/structured-output). |
| **Mostra un esempio** | Per i task a pattern, una coppia input→output vale più di interi paragrafi di descrizione — vedi [Few-Shot](/docs/prompting/few-shot). |
| **Strutturato quando è lungo** | I prompt lunghi hanno bisogno di [tag XML](/docs/prompting/xml-tags) perché Claude distingua le istruzioni dal contenuto. |
| **Protegge dalle risposte inventate** | Quando fornisci un testo sorgente, di' a Claude di usare solo quello — la difesa fondamentale contro le [allucinazioni](/docs/foundations/hallucinations). |
| **Evita qualificatori vaghi** | "buono", "coinvolgente", "professionale" non sono misurabili. Sostituiscili con criteri verificabili. |

:::tip Dal punteggio al rilascio
Un punteggio alto non è l'obiettivo — lo è un risultato affidabile. Tratta la struttura come un punto di partenza, poi mettila alla prova con una [eval](/docs/foundations/evals) reale prima di dipendere dall'output.
:::
