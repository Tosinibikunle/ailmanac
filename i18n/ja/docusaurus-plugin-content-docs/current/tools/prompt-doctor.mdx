すでに書いたプロンプトを貼り付けてください。プロンプトドクターは、Claude にとって重要なプロンプティングの実践に照らしてそれを採点し、どこが弱いかを正確に指摘し、各指摘をそれを解決するページにリンクし、コピーできる改善されたひな形を提供します。

これは**100% ブラウザ内で実行されます** — あなたのプロンプトはどこにも送信されません。チェックは AI 呼び出しではなく、単純なヒューリスティックです。高速で、無料で、透明です。

<PromptDoctor />

## チェック内容

各ルールは、このサイトの詳細なページに対応しています。

| チェック | なぜ重要か |
| --- | --- |
| **行動に移すのに十分か** | 一行のプロンプトは Claude にあなたの意図を推測させます。 |
| **役割を設定する** | 「あなたはシニアの X です」とすると、トーンと判断が鋭く絞り込まれます — [役割](/docs/foundations/roles)を参照。 |
| **明確なタスクを1つ示す** | 先頭の命令動詞（「要約して…」「抽出して…」）は曖昧さを排除します。 |
| **出力の形を指定する** | 形式がないと、Claude が選んだ長さと構造のものを受け取ることになります — [構造化出力](/docs/api/structured-output)を参照。 |
| **例を示す** | パターンタスクでは、1つの入力→出力のペアが段落単位の説明に勝ります — [Few-Shot](/docs/prompting/few-shot)を参照。 |
| **長い場合は構造化する** | 長いプロンプトには [XML タグ](/docs/prompting/xml-tags)が必要で、Claude が指示とコンテンツを区別できるようにします。 |
| **でっち上げの回答を防ぐ** | ソーステキストを提供する場合は、それだけを使うよう Claude に伝えてください — [ハルシネーション](/docs/foundations/hallucinations)に対する中核的な防御策です。 |
| **曖昧な修飾語を避ける** | 「良い」「魅力的な」「プロフェッショナルな」は測定できません。テスト可能な基準に置き換えましょう。 |

:::tip スコアから出荷へ
高スコアが目標ではありません — 信頼できる結果が目標です。ひな形を出発点として扱い、出力に依存する前に実際の[評価](/docs/foundations/evals)でそれを証明してください。
:::
