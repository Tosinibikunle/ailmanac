权限决定了 **Claude Code 在不停下来询问你的情况下能做什么**。调好了，你既有顺畅的工作流又不失掌控；调坏了，你要么对一切机械地放行，要么被询问淹没。

## 三种裁决

每个潜在操作都会归结为以下三者之一：

<Flashcards cards={[{front: "allow（允许）", back: "直接执行，不询问。"}, {front: "ask（询问）", back: "暂停并询问你——这是未被明确允许或拒绝的任何操作的默认行为。"}, {front: "deny（拒绝）", back: "绝不执行，即便被请求也不行。"}]} />

规则通常匹配某个工具加上一个模式，例如允许 `Bash(npm run test:*)`，或拒绝 `Read(./.env)`。

<PromptCard title="允许某个工具 + 模式">{`Bash(npm run test:*)`}</PromptCard>

<PromptCard title="拒绝某个工具 + 模式">{`Read(./.env)`}</PromptCard>

## 权限模式

模式为整个会话设定总体姿态：

| 模式 | 行为 | 适用场景 |
|---|---|---|
| **default / ask** | 对任何未预先允许的操作都询问 | 日常工作 |
| **plan** | 只读；只提方案，从不动手 | 大型/有风险任务——见[规划模式](/docs/claude-code/plan-mode) |
| **acceptEdits** | 自动接受文件编辑 | 一个可信、范围明确的编辑会话 |
| **bypass / yolo** | 完全跳过询问 | 仅限沙箱/CI——**绝不要**在有密钥的机器上用 |

<Callout type="warning" items={["Bypass 只属于沙箱。在你的真实机器上关闭所有询问运行，正是智能体最终碰到它不该碰的东西的方式。请把它留给一次性环境。见“加固自主运行”：/docs/security/hardening-autonomous-runs。"]} />

## 一份合理的初始允许清单

目标：预先允许那些安全、重复的操作；把破坏性操作保持在 **ask** 或 **deny**。

<Steps items={[{title: "允许安全且重复的操作", body: "读取文件，运行你的 test/lint/build 命令，git status/diff。"}, {title: "中等风险的操作先询问", body: "安装依赖，向项目外写文件，网络调用。"}, {title: "拒绝破坏性操作", body: "读取密钥文件（.env、密钥文件），强制推送，rm -rf。"}]} />

把项目规则存到 [`settings.json`](/docs/claude-code/settings)（共享），把个人覆盖项存到 `settings.local.json`。

<Callout type="tip" items={["让它从你的批准中学习：对同一个安全命令批准几次之后，你就会清楚该往允许清单里加什么——把反复出现的询问变成一次性的规则。"]} />
