一个自定义斜杠命令把你反复输入的提示打包成一个 `/word`。定义一次，永久复用（还能分享给你的团队）。

## 它们如何工作

一个命令就是一个 Markdown 文件。把它放进你的命令目录，它的文件名就成了命令。文件的正文就是你调用它时 Claude 运行的提示。

文件的*位置*决定了它的作用范围：

<Steps items={[{title: "项目命令（可共享）", body: "把它保存为 .claude/commands/changelog.md → /changelog。它存在于仓库里，所以整个团队都能用上。"}, {title: "个人命令（所有项目）", body: "把它保存为 ~/.claude/commands/changelog.md → /changelog。它会跟随你出现在本机的每一个项目里。"}, {title: "调用它", body: "在 Claude Code 中输入这个 /word。文件的正文就成了 Claude 运行的提示。"}]} />

<Callout type="tip" items={["决定命令名称的是文件名，而不是前置元数据。changelog.md 永远是 /changelog。"]} />

## 一个最小示例

下面是一个完整的命令文件。前置元数据里放着 `description`；它下面的所有内容就是提示。

<PromptCard title="commit.md —— 起草一条 Conventional Commit">{`---
description: Draft a Conventional Commit message from the staged diff
---

Look at the staged changes with \`git diff --cached\` and write a single
Conventional Commit message (feat/fix/docs/refactor/chore). Output only the
message, no preamble.`}</PromptCard>

把它保存为 `commit.md`，那么 `/commit` 每次都会做这件事。（文件叫什么就给它取什么名字——同样的正文保存为 `changelog.md` 就会是 `/changelog`。）

## 实用的构建块

四种要素把一个静态提示变成一个灵活的命令：

| 构建块 | 它的作用 |
| --- | --- |
| **`$ARGUMENTS`** | 注入用户在命令之后输入的内容。`/review src/auth.ts` → 该路径会落入你的提示。 |
| **运行 bash 并嵌入输出** | 命令可以包含 shell 输出（例如当前的 diff），让 Claude 基于实时状态来行动。 |
| **引用文件** | 指向某些文件以包含它们的内容。 |
| **前置元数据** | `description`（显示在菜单中）以及其他选项。 |

参数、bash 和文件引用的确切语法见[官方文档](https://code.claude.com/docs/en/slash-commands)。

## 命令 vs 技能 vs 子智能体

打包工作有三种方式——区别在于*由谁触发*以及*运行时有多隔离*：

- **斜杠命令**——*由你*按需触发一个已知的工作流。
- **[技能](/docs/claude-code/skills)**——Claude 在相关时*自动*加载专长。
- **[子智能体](/docs/claude-code/subagents)**——把一大块工作委派给一个隔离的智能体。
