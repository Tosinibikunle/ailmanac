粘贴一段你已经写好的提示词。提示词医生会根据对 Claude 真正重要的提示实践为它评分，精确标出薄弱之处，将每条发现链接到能够修复它的页面，并交给你一份可以复制的改进脚手架。

它**100% 在你的浏览器中运行**——你的提示词永远不会被发送到任何地方。这些检查只是简单的启发式规则，而非 AI 调用：快速、免费且透明。

<PromptDoctor />

## 它检查什么

每条规则都对应本站上一个更深入的页面：

| 检查项 | 为什么重要 |
| --- | --- |
| **足以让其行动** | 一行的提示词会迫使 Claude 去猜测你的意图。 |
| **设定角色** | “你是一位资深的 X”能够清晰地聚焦语气和判断力——参见 [角色](/docs/foundations/roles)。 |
| **陈述一个明确的任务** | 一个开头的祈使动词（“总结……”、“提取……”）能消除歧义。 |
| **指定输出形态** | 没有格式，你得到的长度和结构就由 Claude 自行决定——参见 [结构化输出](/docs/api/structured-output)。 |
| **给出示例** | 对于模式类任务，一对输入→输出胜过大段的描述——参见 [少样本](/docs/prompting/few-shot)。 |
| **较长时进行结构化** | 长提示词需要 [XML 标签](/docs/prompting/xml-tags)，这样 Claude 才能区分指令与内容。 |
| **防范编造的答案** | 当你提供源文本时，告诉 Claude 只使用它——这是对抗 [幻觉](/docs/foundations/hallucinations) 的核心防线。 |
| **避免含糊的修饰词** | “好”、“吸引人”、“专业”都无法衡量。请用可检验的标准替换它们。 |

:::tip 从评分到交付
高分并不是目标——可靠的结果才是。把脚手架当作一个起点，然后在依赖输出之前用一次真实的 [评估](/docs/foundations/evals) 来证明它。
:::
