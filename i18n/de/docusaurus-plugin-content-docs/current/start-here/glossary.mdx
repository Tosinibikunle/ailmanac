Definitionen in einfacher Sprache, ohne Vorwissen vorauszusetzen. Überfliege es oder suche nach einem Begriff, auf den du anderswo gestoßen bist.

### Agent
Eine KI, die so eingerichtet ist, dass sie **in einer Schleife Aktionen ausführt** — sie liest ein Ziel, ruft Tools auf, sieht die Ergebnisse und wiederholt das, bis sie fertig ist. Claude Code ist ein Agent für deine Codebasis.

### Anthropic
Das Unternehmen, das Claude entwickelt.

### API (Application Programming Interface)
Eine Möglichkeit für *deine* Software, programmatisch mit Claude zu kommunizieren, anstatt in ein Chatfenster zu tippen. Siehe [Dein erster API-Aufruf](/docs/api/first-call).

### Artifact
Eine lebendige, ausführbare Ausgabe (eine App, ein Diagramm oder ein Dokument), die Claude in einem Seitenpanel auf claude.ai rendert, getrennt vom Chat.

### Chain-of-thought (CoT)
Das Modell zu bitten, **Schritt für Schritt zu denken**, bevor es antwortet, was die Genauigkeit bei schwierigen Problemen verbessert.

### Claude
Die Familie von KI-Modellen (und Apps), die Anthropic herstellt. „Claude" kann je nach Kontext die Chat-App, die Modelle oder die Marke meinen.

### Claude Code
Anthropics **agentisches Kommandozeilen-/IDE-Tool**, das deine Codebasis lesen und bearbeiten, Befehle ausführen und Tools nutzen kann. Siehe [Was Claude Code ist](/docs/claude-code/what-is-claude-code).

### CLAUDE.md
Eine Klartextdatei, in der du Claude Code dauerhafte Anweisungen zu deinem Projekt gibst — seine wirkungsvollste Anpassung.

### Compaction
Das automatische Verdichten älterer Gesprächsabschnitte zu einer kurzen Zusammenfassung, wenn das [Context Window](/docs/foundations/tokens-and-context) voll wird, sodass eine lange Sitzung weiterlaufen kann, ohne den roten Faden zu verlieren. Siehe [Context Management](/docs/claude-code/context-management).

### Context engineering
Bewusst zu entscheiden, was in das [Context Window](/docs/foundations/tokens-and-context) des Modells gelangt — Anweisungen, Beispiele, abgerufene Daten —, um das beste Ergebnis zu erzielen. Die Disziplin hinter Prompting, RAG und Memory.

### Context window
Die maximale Textmenge (gemessen in **Tokens**), die ein Modell auf einmal berücksichtigen kann — dein Prompt *plus* seine Antwort. Wird sie überschritten, fällt der älteste Inhalt heraus.

### Effort
Eine Steuerung bei neueren Claude-Modellen dafür, **wie viel das Modell ungefähr nachdenkt**, bevor es antwortet. Passe die Stufe an die Aufgabe an, statt sie auszureizen — du bezahlst das Nachdenken mit Latenz und Kosten. Siehe [Extended Thinking & Effort](/docs/api/thinking-and-effort).

### Embedding
Eine Liste von Zahlen, die die *Bedeutung* eines Textstücks repräsentiert, sodass Software Dinge finden kann, die in der Bedeutung ähnlich sind (nicht nur passende Schlüsselwörter). Treibt die semantische Suche und RAG an.

### Eval (Evaluation)
Ein wiederholbarer Test der Qualität von KI-Ausgaben, damit du erkennen kannst, ob eine Änderung die Dinge besser oder schlechter gemacht hat.

### Few-shot
Ein **paar durchgearbeitete Beispiele** in deinen Prompt aufzunehmen, um dem Modell das gewünschte Muster zu zeigen.

### Fine-tuning
Ein Modell mit deinen eigenen Beispielen weiter zu trainieren, um es zu spezialisieren. Meist ein letztes Mittel — probiere [zuerst Prompting und RAG](/docs/foundations).

### Grounding
Das Modell anzuweisen, **nur aus dem von dir bereitgestellten Kontext** zu antworten — und „Ich weiß es nicht" zu sagen, wenn die Antwort dort nicht steht. Die wichtigste Verteidigung auf Prompt-Ebene gegen Halluzinationen. Siehe [die Pattern Library](/docs/prompting/pattern-library).

### Guardrails
Regeln und Grenzen, die das Verhalten einer KI sicher und auf Kurs halten — z. B. Berechtigungsregeln, „vor dem Fertigstellen Tests ausführen" oder menschliche Freigabe für riskante Aktionen. Siehe [Agenten absichern](/docs/security/securing-agents).

### Hallucination
Wenn ein Modell etwas **Falsches, aber Selbstsicheres** behauptet. Die zentrale Fähigkeit ist Verifikation. Siehe [Halluzinationen](/docs/foundations).

### Hook
In Claude Code ein Befehl, der **automatisch** bei einem Lebenszyklus-Ereignis ausgeführt wird (z. B. nach jeder Dateibearbeitung), um Formatierung, Kontrollen oder Sicherheit durchzusetzen.

### Inference
Der Vorgang, ein Modell auszuführen, um eine Ausgabe zu erhalten (im Gegensatz zum Trainieren).

### Latency
Wie lange du auf eine Antwort wartest. Größere Modelle sind in der Regel langsamer; [Streaming](/docs/api/streaming) verbessert die *gefühlte* Latenz, indem die Ausgabe schon während der Generierung angezeigt wird.

### LLM (Large Language Model)
Die Art von KI hinter Claude: trainiert, um **den nächsten Textabschnitt vorherzusagen**, was sich als ausreichend erweist, um zu schreiben, zu programmieren, zusammenzufassen und zu schlussfolgern. Siehe [Was ist ein LLM?](/docs/foundations).

### MCP (Model Context Protocol)
Ein **offener Standard** zum Verbinden von KI mit externen Tools und Daten (Datenbanken, GitHub, Browser). Ein „MCP-Server" stellt ein Tool bereit; Claude kann es aufrufen. Siehe [MCP](/docs/claude-code).

### Multimodal
Ein Modell, das **mehr als Text** verarbeitet — z. B. Bilder und PDFs als Eingabe.

### Output style
In Claude Code eine Voreinstellung, die ändert, **wie Claude kommuniziert** (z. B. knapp, erklärend, lernend), ohne zu ändern, was es kann. Siehe [Output Styles](/docs/claude-code/output-styles).

### Permission mode
In Claude Code, wie viel es **ohne nachzufragen** tun darf — vom Nachfragen bei jeder Aktion bis zum automatischen Annehmen von Bearbeitungen. Der zentrale Regler zwischen Geschwindigkeit und Sicherheit. Siehe [Berechtigungen](/docs/claude-code/permissions).

### Plan mode
Ein schreibgeschützter Claude-Code-Modus, der recherchiert und einen Plan vorschlägt, **bevor** er Dateien anrührt — du gibst dein Einverständnis, bevor er handelt. Siehe [Plan Mode](/docs/claude-code/plan-mode).

### Prompt
Was du dem Modell sendest: deine Anweisungen, deine Frage und jeglichen Kontext.

### Prompt caching
Das Wiederverwenden des unveränderten vorderen Teils eines Prompts über mehrere Aufrufe hinweg, um Kosten und Latenz zu senken. Siehe [Prompt Caching](/docs/api).

### Prompt injection
Ein Angriff, bei dem **bösartige Anweisungen, die in Inhalten versteckt sind**, die die KI liest (eine Webseite, ein Dokument), versuchen, sie zu übernehmen. Siehe [Prompt Injection](/docs/security).

### RAG (Retrieval-Augmented Generation)
Relevante Ausschnitte aus *deinen* Daten abrufen und sie dem Prompt hinzufügen, damit das Modell darüber Antworten gibt. Siehe [RAG](/docs/foundations).

### Reasoning / extended thinking
Ein Modus, in dem das Modell vor dem Antworten zusätzliche Rechenleistung zum „Nachdenken" aufwendet, für schwierigere Aufgaben. Bei neueren Modellen stellst du ihn über [Effort](/docs/api/thinking-and-effort) ein.

### Skill
Eine verpackte Fähigkeit (Anweisungen + optionale Skripte), die Claude **bei Bedarf** lädt, wenn sie relevant ist. Siehe [Skills](/docs/claude-code).

### Slash command
Eine mit `/` beginnende Abkürzung in Claude Code — eingebaut (wie `/clear`) oder dein eigener wiederverwendbarer Prompt —, die eine definierte Aktion auslöst. Siehe [Slash Commands](/docs/claude-code/slash-commands).

### Structured output
Ergebnisse in einer strengen, maschinenlesbaren Form zu erhalten (z. B. JSON, das einem Schema entspricht) statt als freier Text — sodass andere Software sich darauf verlassen kann. Siehe [Strukturierte Ausgabe](/docs/api/structured-output).

### Subagent
Eine separate Claude-Instanz mit eigenem Kontext und eigenen Tools, die genutzt wird, um einen Teil einer Aufgabe zu **delegieren** (z. B. Recherche), damit die Hauptsitzung fokussiert bleibt.

### System prompt
Die übergeordneten Anweisungen, die die Rolle und die Regeln des Modells für eine gesamte Unterhaltung festlegen — dein mächtigster Steuerungshebel.

### Temperature
Eine Einstellung (bei manchen Modellen), die die Zufälligkeit steuert: niedrig = fokussiert und reproduzierbar, hoch = abwechslungsreich und kreativ.

### Token
Die Einheit, die Modelle lesen und zählen — etwa ¾ eines Wortes im Englischen. Preise und Kontextlimits werden in Tokens gemessen. Siehe [Tokens & Kontext](/docs/foundations).

### Tool use / function calling
Dem Modell zu erlauben, von dir definierte Funktionen aufzurufen (Suche, Taschenrechner, deine API) und die Ergebnisse zu nutzen. Siehe [Tool Use](/docs/api).
