Collez un prompt que vous avez déjà rédigé. Le Prompt Doctor l'évalue selon les pratiques de prompting qui comptent pour Claude, signale précisément ce qui est faible, relie chaque constat à la page qui le corrige et vous remet une trame améliorée à copier.

Il fonctionne **100 % dans votre navigateur** — votre prompt n'est jamais envoyé nulle part. Les vérifications sont de simples heuristiques, et non un appel à une IA : rapides, gratuites et transparentes.

<PromptDoctor />

## Ce qu'il vérifie

Chaque règle renvoie à une page plus approfondie de ce site :

| Vérification | Pourquoi c'est important |
| --- | --- |
| **Assez d'éléments pour agir** | Les prompts d'une seule ligne obligent Claude à deviner votre intention. |
| **Définit un rôle** | « Tu es un X senior » concentre nettement le ton et le jugement — voir [Rôles](/docs/foundations/roles). |
| **Énonce une seule tâche claire** | Un verbe à l'impératif en tête (« Résume… », « Extrais… ») lève toute ambiguïté. |
| **Précise le format de sortie** | Sans format, vous obtenez la longueur et la structure que Claude choisit — voir [Sortie structurée](/docs/api/structured-output). |
| **Montre un exemple** | Pour les tâches à motif, une paire entrée→sortie vaut mieux que des paragraphes de description — voir [Few-Shot](/docs/prompting/few-shot). |
| **Structuré quand c'est long** | Les prompts longs ont besoin de [balises XML](/docs/prompting/xml-tags) pour que Claude distingue les instructions du contenu. |
| **Se prémunit contre les réponses inventées** | Quand vous fournissez un texte source, dites à Claude de n'utiliser que celui-ci — la défense centrale contre les [hallucinations](/docs/foundations/hallucinations). |
| **Évite les qualificatifs vagues** | « bon », « engageant », « professionnel » ne sont pas mesurables. Remplacez-les par des critères vérifiables. |

:::tip Du score à la mise en production
Un score élevé n'est pas l'objectif — un résultat fiable l'est. Considérez la trame comme un point de départ, puis validez-la avec une vraie [éval](/docs/foundations/evals) avant de dépendre de la sortie.
:::
