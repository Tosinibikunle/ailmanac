Cuando un prompt mezcla instrucciones, un documento, ejemplos y una pregunta, el modelo puede confundirlos entre sí. Las **etiquetas de estilo XML** son una forma limpia de etiquetar cada parte — y Claude responde especialmente bien a ellas.

## La idea

Envuelve cada sección en una etiqueta con nombre para que quede claro qué es cada cosa:

<PromptCard title="Estructura de prompt etiquetada">{`<instructions>
Summarize the document for a busy executive. Use only the document; if a fact
isn't there, say so.
</instructions>

<document>
{paste the long document here}
</document>

<format>
3 bullet points, then a one-line "decision needed".
</format>`}</PromptCard>

Las etiquetas son simplemente texto que tú inventas — `<document>`, `<example>`, `<context>`, `<rules>` — pero le dan al modelo límites nítidos.

## Por qué ayuda

<Callout type="tip" items={["Separa los datos de las instrucciones — el modelo es menos propenso a obedecer texto extraviado dentro de un documento pegado (una defensa leve contra la inyección de prompts — ver /docs/security/prompt-injection).", "Reduce el \"ignoró parte de mi prompt\". Cada parte está claramente delimitada.", "Facilita el análisis de las salidas — puedes pedir a Claude que ponga su respuesta en etiquetas <answer> y extraerla de forma fiable.", "Se combina con few-shot (/docs/prompting/few-shot) — envuelve cada ejemplo en <example>."]} />

Las cuatro ventajas en detalle:

- **Separa los datos de las instrucciones** — el modelo es menos propenso a obedecer texto extraviado dentro de un documento pegado (una defensa leve contra la [inyección de prompts](/docs/security/prompt-injection)).
- **Reduce el "ignoró parte de mi prompt".** Cada parte está claramente delimitada.
- **Facilita el análisis de las salidas** — puedes pedir a Claude que ponga su respuesta en etiquetas `<answer>` y extraerla de forma fiable.
- **Se combina con [few-shot](/docs/prompting/few-shot)** — envuelve cada ejemplo en `<example>`.

## Pedir salida etiquetada

<Steps items={[{title: "Pide a Claude que envuelva su salida en etiquetas", body: "Dile al modelo exactamente qué etiquetas usar para el razonamiento frente a la respuesta final."}, {title: "Extrae solo la parte que necesitas", body: "Tu código puede tomar únicamente el contenido de <answer>, ignorando el razonamiento."}, {title: "Combínalo con la salida estructurada", body: "Funciona bien junto a la salida estructurada (/docs/api/structured-output) cuando necesitas resultados legibles por máquina."}]} />

<PromptCard title="Solicitar salida etiquetada">{`Put your reasoning in <thinking> tags and your final answer in <answer> tags.`}</PromptCard>

Luego tu código puede tomar solo el contenido de `<answer>`. Combina bien con la [salida estructurada](/docs/api/structured-output) cuando necesitas resultados legibles por máquina.

## Consejos

<Callout type="takeaways" items={["Sé consistente — abre y cierra cada etiqueta; reutiliza los mismos nombres.", "Nombra las etiquetas con sentido (<contract>, no <x>).", "No abuses de las etiquetas en prompts triviales — usa esto cuando realmente haya varias partes distintas."]} />

- **Sé consistente** — abre y cierra cada etiqueta; reutiliza los mismos nombres.
- **Nombra las etiquetas con sentido** (`<contract>`, no `<x>`).
- **No abuses de las etiquetas** en prompts triviales — usa esto cuando realmente haya varias partes distintas.
