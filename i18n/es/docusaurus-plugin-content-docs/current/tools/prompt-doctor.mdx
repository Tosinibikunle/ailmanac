Pega un prompt que ya hayas escrito. El Prompt Doctor lo puntúa frente a las prácticas de prompting que importan para Claude, señala exactamente qué es débil, enlaza cada hallazgo con la página que lo corrige y te entrega un esqueleto mejorado para copiar.

Se ejecuta **100% en tu navegador** — tu prompt nunca se envía a ningún sitio. Las comprobaciones son simples heurísticas, no una llamada a una IA: rápidas, gratuitas y transparentes.

<PromptDoctor />

## Qué comprueba

Cada regla se corresponde con una página más detallada de este sitio:

| Comprobación | Por qué importa |
| --- | --- |
| **Suficiente para actuar** | Los prompts de una sola línea obligan a Claude a adivinar tu intención. |
| **Establece un rol** | "Eres un X senior" enfoca con precisión el tono y el criterio — consulta [Roles](/docs/foundations/roles). |
| **Plantea una tarea clara** | Un verbo imperativo inicial ("Resume…", "Extrae…") elimina la ambigüedad. |
| **Especifica el formato de salida** | Sin un formato obtienes la longitud y estructura que Claude elija — consulta [Salida estructurada](/docs/api/structured-output). |
| **Muestra un ejemplo** | Para tareas de patrón, un par entrada→salida supera a párrafos de descripción — consulta [Few-Shot](/docs/prompting/few-shot). |
| **Estructurado cuando es largo** | Los prompts largos necesitan [etiquetas XML](/docs/prompting/xml-tags) para que Claude distinga las instrucciones del contenido. |
| **Protege contra respuestas inventadas** | Cuando aportas texto fuente, dile a Claude que use solo ese texto — la defensa principal contra las [alucinaciones](/docs/foundations/hallucinations). |
| **Evita calificativos vagos** | "bueno", "atractivo", "profesional" no son medibles. Sustitúyelos por criterios comprobables. |

:::tip De la puntuación a la entrega
Una puntuación alta no es el objetivo — un resultado fiable sí lo es. Trata el esqueleto como un punto de partida y luego demuéstralo con una [eval](/docs/foundations/evals) real antes de depender de la salida.
:::
