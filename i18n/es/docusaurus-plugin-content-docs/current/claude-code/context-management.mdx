## La idea central

Cada sesión tiene una [ventana de contexto](/docs/foundations/tokens-and-context) finita. A medida que se llena, el contenido más antiguo queda comprimido y Claude puede empezar a "derivar". Gestionar el contexto de forma deliberada es lo que mantiene afiladas las sesiones largas.

## Los dos comandos esenciales

Dos comandos cubren casi todas las situaciones. El truco está en hacer coincidir el comando con tu intención.

<Flashcards cards={[{front: "/clear", back: "Borra la conversación y empieza de cero. Úsalo cuando cambias a una tarea NO RELACIONADA. La forma más barata de recuperar un contexto limpio y enfocado."}, {front: "/compact", back: "Resume la conversación hasta el momento en una forma compacta — conserva la esencia mientras libera espacio. Úsalo cuando quieres CONTINUAR la misma tarea pero la ventana se está llenando."}]} />

Decídelo con una sola pregunta: ¿es una tarea nueva o la misma?

```mermaid
flowchart TD
  A{New task or same task?} -->|New / unrelated| B[/clear]
  A -->|Same task, context full| C[/compact]
```

## Compactación automática

Cuando el contexto se acerca al límite, Claude Code puede **compactar automáticamente** para que el trabajo continúe sin una parada en seco. Es una red de seguridad — pero hacer `/clear` de forma proactiva entre tareas da mejores resultados que depender de ella.

<Callout type="tip" items={["Trata la compactación automática como un respaldo, no como una estrategia. Limpiar entre tareas no relacionadas mantiene el contexto más limpio que dejar que la ventana se llene hasta que se active la compactación automática."]} />

## Técnicas para mantener la eficacia

Más allá de los dos comandos, cuatro hábitos mantienen tu contexto ligero desde el principio.

Estos se corresponden con la documentación viva: [subagentes](/docs/claude-code/subagents), [CLAUDE.md](/docs/claude-code/claude-md) y [statusline](/docs/claude-code/statusline).

:::tip Síntoma → solución
Si Claude empieza a olvidar una instrucción anterior o a repetirse, lo más probable es que la ventana esté llena. `/compact` (misma tarea) o `/clear` (tarea nueva) suele arreglarlo al instante. Más en [Resolución de problemas](/docs/contribute/troubleshooting).
:::
