تقرر الأذونات **ما الذي يستطيع Claude Code فعله دون أن يتوقف ليسألك**. اضبطها جيدًا فتحصل على انسيابية دون فقدان التحكم؛ واضبطها بشكل سيئ فإما أن تختم على كل شيء دون تفكير أو تغرق في المطالبات.

## الأحكام الثلاثة

كل إجراء محتمل يؤول إلى أحد التالي:

<Flashcards cards={[{front: "السماح (allow)", back: "نفّذه، بلا مطالبة."}, {front: "السؤال (ask)", back: "توقّف واسألك — الافتراضي لأي شيء غير مسموح أو مرفوض صراحةً."}, {front: "الرفض (deny)", back: "أبدًا، حتى لو طُلب."}]} />

تطابق القواعد عادةً أداةً مع نمط، مثلًا السماح بـ `Bash(npm run test:*)` أو رفض `Read(./.env)`.

## أوضاع الأذونات

يحدد الوضع الموقف العام للجلسة:

| الوضع | السلوك | استخدمه عندما |
|---|---|---|
| **default / ask** | يطالب لأي شيء غير مسموح مسبقًا | العمل اليومي |
| **plan** | قرائي فقط؛ يقترح ولا يفعل أبدًا | المهام الكبيرة/المحفوفة بالمخاطر — راجع [وضع التخطيط](/docs/claude-code/plan-mode) |
| **acceptEdits** | يقبل تعديلات الملفات تلقائيًا | جلسة تحرير موثوقة ومحددة النطاق جيدًا |
| **bypass / yolo** | يتخطى المطالبات تمامًا | الصناديق الرملية/CI فقط — **أبدًا** على جهاز يحوي أسرارًا |

<Callout type="warning" items={["التجاوز ينتمي إلى الصندوق الرملي. التشغيل مع إيقاف كل المطالبات على جهازك الحقيقي هو الطريقة التي ينتهي بها الوكيل إلى لمس شيء لا ينبغي له. احتفظ به للبيئات القابلة للتخلص منها. راجع تحصين عمليات التشغيل المستقلة في /docs/security/hardening-autonomous-runs."]} />

## قائمة سماح مبدئية معقولة

الهدف: اسمح مسبقًا بالأشياء الآمنة والمتكررة؛ وأبقِ الأشياء المدمّرة على **السؤال** أو **الرفض**.

<Steps items={[{title: "اسمح بالآمن + المتكرر", body: "قراءة الملفات، تشغيل أوامر الاختبار/الفحص/البناء، git status/diff."}, {title: "اسأل قبل متوسط المخاطر", body: "تثبيت الاعتماديات، كتابة ملفات خارج المشروع، نداءات الشبكة."}, {title: "ارفض المدمّر", body: "قراءة ملفات الأسرار (.env، ملفات المفاتيح)، الدفع القسري (force-push)، rm -rf."}]} />

خزّن قواعد المشروع في [`settings.json`](/docs/claude-code/settings) (المشتركة) والتجاوزات الشخصية في `settings.local.json`.
