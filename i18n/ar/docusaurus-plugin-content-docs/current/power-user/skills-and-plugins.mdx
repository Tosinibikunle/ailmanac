أسرع طريقة لترتقي بمستوى Claude Code هي أن تتوقف عن كتابة كل شيء يدويًا و**تثبّت قدرات أتقنها آخرون بالفعل**. تمنح المهارات Claude خبرة عند الطلب؛ أما الإضافات فتجمع المهارات والوكلاء والأوامر والخطافات وخوادم MCP في وحدة واحدة قابلة للإصدار والتثبيت. هذه الصفحة قائمة قصيرة منسّقة وجاهزة للتثبيت — ما الذي يفعله كل عنصر، ومتى تلجأ إليه، والأمر الدقيق لإضافته.

إن كانت آلية العمل غير واضحة لديك، فاقرأ أولًا [المهارات](/docs/claude-code/skills) و[الإضافات والأسواق](/docs/claude-code/plugins-marketplaces) — هذه الصفحة تفترض أنك تعرف ما هو ملف `SKILL.md`.

<VerifyNote lastVerified="2026-06-24" source="https://code.claude.com/docs/en/discover-plugins">
تتغيّر أسماء الأسواق، وكتالوج الإضافات الرسمية، وأعداد نجوم مستودعات المجتمع بشكل متكرّر. تنفّذ الإضافات تعليمات برمجية بصلاحياتك — لا تثبّت إلا من مصادر تثق بها، وتأكّد من الأوامر الحالية في الوثائق الرسمية قبل الاعتماد عليها.
</VerifyNote>

## المهارة مقابل الإضافة: النموذج في 20 ثانية

**المهارة** هي مجلد يحتوي على `SKILL.md` (واجهة أمامية فيها `description` + تعليمات). يحمّلها Claude فقط عندما تكون ذات صلة، أو تستدعيها أنت عبر `/skill-name`. تعيش في `~/.claude/skills/` (شخصية)، أو في `.claude/skills/` الخاص بمشروع، أو داخل إضافة. وفقًا للوثائق الرسمية، فقد دُمجت أوامر الشرطة المائلة المخصّصة في المهارات — فكلٌّ من `.claude/commands/deploy.md` و`.claude/skills/deploy/SKILL.md` ينشئ الأمر `/deploy`.

**الإضافة** حزمة قابلة للتوزيع يمكنها أن تشحن المهارات والوكلاء والخطافات وخوادم MCP وخوادم LSP معًا، مع بيان `.claude-plugin/plugin.json`. تحصل على الإضافات من **سوق** — كتالوج تضيفه مرة واحدة ثم تثبّت منه. تُسبَق مهارات الإضافات بمساحة اسم، مثل `/commit-commands:commit`.

## أضف سوقًا، وثبّت إضافة

سوق Anthropic الرسمي (`claude-plugins-official`) متاح تلقائيًا. وهناك سوقان آخران يستحقان الإضافة يدويًا: سوق **المجتمع** المنسّق، وسوق **العروض التجريبية** الذي يضم إضافات أمثلة.

## إضافات رسمية تستحق التثبيت أولًا

تأتي هذه ضمن سوق Anthropic الرسمي (`claude-plugins-official`) وهي نقطة الانطلاق الأعلى قيمة. ثبّتها عبر `/plugin install <name>@claude-plugins-official`.

### ذكاء التعليمات البرمجية (LSP) — اجعل Claude يرى أخطاء الأنواع فورًا

تربط إضافات LSP خادم لغة مع Claude Code، فيحصل Claude بعد كل تعديل على تشخيصات تلقائية (أخطاء الأنواع، الاستيرادات المفقودة) وتنقّل دقيق (الانتقال إلى التعريف، إيجاد المراجع) بدلًا من تخمينات grep. وتتطلّب وجود ثنائي خادم اللغة المطابق على جهازك.

- **متى تستخدمها:** أي جلسة برمجة غير تافهة بلغة ذات أنواع. هذا على الأرجح أكبر ترقية دقّة منفردة لأعمال التطوير.
- **المقايضة:** يمكن لخوادم مثل `rust-analyzer` و`pyright` أن تستهلك قدرًا كبيرًا من الذاكرة في المستودعات الكبيرة؛ عطّلها إن أزعجتك.

<PromptCard title="ثبّت LSP لحزمتك التقنية">
{`/plugin install typescript-lsp@claude-plugins-official   # needs: typescript-language-server
/plugin install pyright-lsp@claude-plugins-official       # needs: pyright-langserver
/plugin install rust-analyzer-lsp@claude-plugins-official # needs: rust-analyzer
/plugin install gopls-lsp@claude-plugins-official         # needs: gopls`}
</PromptCard>

### سير عمل Git وطلبات السحب

- **`commit-commands`** — سير عمل git: التجهيز، توليد رسالة الالتزام، الدفع، فتح طلب سحب. استخدمها عندما تريد التزامات نظيفة ومتّسقة دون كتابة المراسم. المهارات مسبوقة بمساحة اسم، مثل `/commit-commands:commit`.
- **`pr-review-toolkit`** — وكلاء متخصّصون لمراجعة طلبات السحب. استخدمها قبل الدمج، أو للحصول على نظرة ثانية على فرق التغييرات.

<PromptCard title="أساسيات Git/طلبات السحب">
{`/plugin install commit-commands@claude-plugins-official
/plugin install pr-review-toolkit@claude-plugins-official
/reload-plugins`}
</PromptCard>

### مراجعة أمنية تلقائية

- **`security-guidance`** — تراجع كل تغيير يجريه Claude بحثًا عن الثغرات الشائعة وتوجّه Claude لإصلاحها في الجلسة نفسها. استخدمها كشبكة أمان دائمة في أي مشروع يكتب تعليمات برمجية؛ فهي تكمّل مراجعة حقيقية (ولا تحل محلها).

<PromptCard title="تمرير أمني دائم التشغيل">
{`/plugin install security-guidance@claude-plugins-official`}
</PromptCard>

### تكاملات خارجية (MCP، مُهيّأة مسبقًا)

تجمع هذه [خادم MCP](/docs/claude-code/mcp) جاهزًا للاستخدام فتتخطّى الإعداد اليدوي. الجأ إلى واحدة منها عندما تريد أن يتصرّف Claude مباشرة في تلك الخدمة.

- **التحكم في المصدر:** `github`، `gitlab`
- **إدارة المشاريع:** `linear`، `notion`، `asana`، `atlassian` (Jira/Confluence)
- **البنية التحتية:** `vercel`، `firebase`، `supabase`
- **التصميم / التواصل / المراقبة:** `figma`، `slack`، `sentry`

<PromptCard title="اربط Claude بأدواتك">
{`/plugin install github@claude-plugins-official
/plugin install linear@claude-plugins-official
/plugin install supabase@claude-plugins-official`}
</PromptCard>

### أدوات بناء خاصة بك وأنماط مخرجات

- **`plugin-dev`** — مجموعة أدوات لتأليف إضافاتك الخاصة. استخدمها فور أن يصبح إعداد `.claude/` لديك جديرًا بالمشاركة.
- **`agent-sdk-dev`** — أدوات للبناء على Claude Agent SDK.
- **`explanatory-output-style` / `learning-output-style`** — تغيّر *كيفية* استجابة Claude: شروحات تُعلِّم أثناء العمل، أو وضع تعلّم تفاعلي. ممتازة للتأهيل أو عندما تريد أن تفهم التغييرات فعلًا.

## مهارات عالية القيمة حسب المهمة المطلوب إنجازها

تتألّق المهارات في المهام المتكرّرة المحمّلة بالسياق. تنشر Anthropic مجموعة مرجعية عامة في مستودع [`anthropics/skills`](https://github.com/anthropics/skills) — وهي الأنماط نفسها التي تدعم قدرات Claude المستندية والإبداعية. يمكنك نسخ مجلد مهارة إلى `~/.claude/skills/` (شخصية) أو `.claude/skills/` (مشروع) فتُحمَّل في الجلسة التالية.

| المهمة | المهارة (من anthropics/skills) | متى تلجأ إليها |
| :-- | :-- | :-- |
| **المستندات** | `docx`، `pdf`، `pptx`، `xlsx` | توليد أو تحرير ملفات Office/PDF حقيقية — تقارير وعروض وجداول بيانات ونماذج قابلة للتعبئة. (تطبيقات مرجعية متاحة المصدر.) |
| **بناء الواجهات** | `web-artifacts-builder` / `artifacts-builder` | إنتاج أدوات HTML/JS مستقلّة وتطبيقات مصغّرة. |
| **توسعة Claude** | `mcp-builder` | إنشاء هيكل خادم MCP جديد عندما لا يوجد تكامل بعد. |
| **العلامة والتواصل** | مهارات نمط brand-guidelines / internal-comms | الحفاظ على النبرة والتسمية والتنسيق متوافقًا مع العلامة عبر المخرجات. |

<Callout type="tip" items={["لا تُلقِ 30 مهارة في ~/.claude/skills دفعة واحدة. المهارات تعمل بالكشف التدريجي — لا يُحمَّل سوى الاسم + الوصف حتى يتم التحفيز — لكن كومة منتفخة وموصوفة بغموض تجعل التحفيز غير موثوق. ثبّت القليل، واكتب أوصاف \"استخدمها عندما…\" حادّة، وانمُ بتعمّد.", "تشحن AILmanac حزم مهارات جاهزة يمكنك إسقاطها لتجربتها — انظر /docs/templates/skills."]} />

### مهارة ذاتية التحفيز يمكنك لصقها الآن

للأعمال كثيفة البحث، فإن مهارة تُرسِّخ Claude في مصادر حديثة أفضل من إعادة شرح منهجك في كل مرة. أسقط هذا في `~/.claude/skills/research-brief/SKILL.md`:

<PromptCard title="~/.claude/skills/research-brief/SKILL.md">
{`---
name: research-brief
description: Use when the user asks for a researched brief, literature scan, or "what's the current state of X" with citations.
---

# Research brief

When asked for a brief on a topic:
1. Search broadly, then narrow to authoritative/primary sources.
2. Synthesize in your own words — never paste source text verbatim.
3. Flag uncertainty and date-sensitive claims explicitly.
4. End with a "Sources" list of real, clickable URLs you actually used.`}
</PromptCard>
