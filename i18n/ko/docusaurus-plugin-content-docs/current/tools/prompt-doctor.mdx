이미 작성한 프롬프트를 붙여넣으세요. 프롬프트 닥터는 Claude에게 중요한 프롬프트 작성 사례를 기준으로 점수를 매기고, 어디가 약한지 정확히 짚어내며, 각 진단을 해결책이 담긴 페이지로 연결하고, 복사해서 쓸 수 있는 개선된 스캐폴드를 건네줍니다.

이 도구는 **100% 브라우저에서** 실행됩니다 — 프롬프트가 어디로도 전송되지 않습니다. 검사는 AI 호출이 아니라 단순한 휴리스틱이라 빠르고, 무료이며, 투명합니다.

<PromptDoctor />

## 무엇을 검사하나요

각 규칙은 이 사이트의 더 자세한 페이지로 연결됩니다.

| 검사 항목 | 왜 중요한가 |
| --- | --- |
| **행동에 옮길 만큼 충분한가** | 한 줄짜리 프롬프트는 Claude가 의도를 추측하게 만듭니다. |
| **역할을 부여하는가** | "당신은 시니어 X입니다"는 어조와 판단을 날카롭게 집중시킵니다 — [역할](/docs/foundations/roles)을 참고하세요. |
| **하나의 명확한 작업을 제시하는가** | 앞에 오는 명령형 동사("요약하라…", "추출하라…")는 모호함을 없앱니다. |
| **출력 형태를 지정하는가** | 형식이 없으면 Claude가 고른 길이와 구조를 그대로 받게 됩니다 — [구조화된 출력](/docs/api/structured-output)을 참고하세요. |
| **예시를 보여주는가** | 패턴 작업에서는 입력→출력 한 쌍이 여러 문단의 설명보다 낫습니다 — [퓨샷](/docs/prompting/few-shot)을 참고하세요. |
| **길 때 구조화하는가** | 긴 프롬프트에는 Claude가 지시와 내용을 구분할 수 있도록 [XML 태그](/docs/prompting/xml-tags)가 필요합니다. |
| **지어낸 답변을 방지하는가** | 원본 텍스트를 제공할 때는 그것만 사용하라고 Claude에게 지시하세요 — [환각](/docs/foundations/hallucinations)에 대한 핵심 방어책입니다. |
| **모호한 수식어를 피하는가** | "좋은", "매력적인", "전문적인"은 측정할 수 없습니다. 검증 가능한 기준으로 바꾸세요. |

:::tip 점수에서 출시까지
높은 점수가 목표는 아닙니다 — 신뢰할 수 있는 결과가 목표입니다. 스캐폴드는 출발점으로 삼고, 출력에 의존하기 전에 실제 [평가](/docs/foundations/evals)로 검증하세요.
:::
