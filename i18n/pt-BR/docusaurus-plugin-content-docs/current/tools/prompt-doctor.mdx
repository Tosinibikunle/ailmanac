<LevelBadge level="all" />

Cole um prompt que você já escreveu. O Prompt Doctor avalia o prompt em relação às práticas de prompting que importam para o Claude, sinaliza exatamente o que está fraco, vincula cada constatação à página que corrige o problema e te entrega um esqueleto aprimorado para copiar.

Ele roda **100% no seu navegador** — seu prompt nunca é enviado para lugar nenhum. As verificações são heurísticas simples, não uma chamada de IA: rápidas, gratuitas e transparentes.

<PromptDoctor />

## O que ele verifica

Cada regra remete a uma página mais aprofundada neste site:

| Verificação | Por que importa |
| --- | --- |
| **Conteúdo suficiente para agir** | Prompts de uma linha forçam o Claude a adivinhar sua intenção. |
| **Define um papel** | "Você é um X sênior" foca de forma nítida o tom e o julgamento — veja [Papéis](/docs/foundations/roles). |
| **Declara uma única tarefa clara** | Um verbo imperativo inicial ("Resuma…", "Extraia…") elimina a ambiguidade. |
| **Especifica o formato de saída** | Sem um formato, você recebe qualquer extensão e estrutura que o Claude escolher — veja [Saída Estruturada](/docs/api/structured-output). |
| **Mostra um exemplo** | Para tarefas de padrão, um par entrada→saída vale mais do que parágrafos de descrição — veja [Few-Shot](/docs/prompting/few-shot). |
| **Estruturado quando longo** | Prompts longos precisam de [tags XML](/docs/prompting/xml-tags) para que o Claude consiga distinguir instruções de conteúdo. |
| **Protege contra respostas inventadas** | Quando você fornece um texto-fonte, diga ao Claude para usar apenas ele — a defesa central contra a [alucinação](/docs/foundations/hallucinations). |
| **Evita qualificadores vagos** | "bom", "envolvente", "profissional" não são mensuráveis. Substitua-os por critérios testáveis. |

:::tip Do placar à entrega
Uma nota alta não é o objetivo — um resultado confiável é. Trate o esqueleto como um ponto de partida e então comprove-o com uma [eval](/docs/foundations/evals) real antes de depender da saída.
:::
