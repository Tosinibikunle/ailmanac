Um comando slash personalizado empacota um prompt que você fica redigitando em um único `/palavra`. Defina-o uma vez; reutilize-o para sempre (e compartilhe-o com sua equipe).

## Como funcionam

Um comando é apenas um arquivo Markdown. Coloque-o no seu diretório de comandos e o nome do arquivo se torna o comando. O corpo do arquivo é o prompt que o Claude executa quando você o invoca.

A *localização* do arquivo decide o seu escopo:

## Um exemplo mínimo

Aqui está um arquivo de comando completo. O frontmatter contém a `description`; tudo abaixo dele é o prompt.

<PromptCard title="commit.md — redigir um Conventional Commit">{`---
description: Draft a Conventional Commit message from the staged diff
---

Look at the staged changes with \`git diff --cached\` and write a single
Conventional Commit message (feat/fix/docs/refactor/chore). Output only the
message, no preamble.`}</PromptCard>

Salve isso como `commit.md` e `/commit` faz isso toda vez. (Dê a ele o nome que o arquivo tiver — o mesmo corpo salvo como `changelog.md` seria `/changelog`.)

## Os blocos de construção úteis

Quatro ingredientes transformam um prompt estático em um comando flexível:

| Bloco de construção | O que faz |
| --- | --- |
| **`$ARGUMENTS`** | Injeta o que o usuário digitou após o comando. `/review src/auth.ts` → o caminho cai no seu prompt. |
| **Execute bash e incorpore a saída** | Os comandos podem incluir a saída de shell (por exemplo, o diff atual) para que o Claude aja sobre o estado ao vivo. |
| **Referencie arquivos** | Aponte para arquivos para incluir seu conteúdo. |
| **Frontmatter** | `description` (exibida no menu) e outras opções. |

Veja a [documentação oficial](https://code.claude.com/docs/en/slash-commands) para a sintaxe exata de argumentos, bash e referências de arquivo.

## Comando vs skill vs subagente

Três formas de empacotar trabalho — a diferença está em *quem o aciona* e *quão isolado ele roda*:

- **Comando slash** — *você* aciona um fluxo de trabalho conhecido sob demanda.
- **[Skill](/docs/claude-code/skills)** — o Claude carrega expertise *automaticamente* quando relevante.
- **[Subagente](/docs/claude-code/subagents)** — delega uma parte do trabalho a um agente isolado.
