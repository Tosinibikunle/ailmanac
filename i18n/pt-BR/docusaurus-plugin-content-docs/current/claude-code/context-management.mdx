## A grande ideia

Toda sessão tem uma [janela de contexto](/docs/foundations/tokens-and-context) finita. Conforme ela enche, o conteúdo mais antigo é comprimido e o Claude pode começar a "desviar". Gerenciar o contexto de forma deliberada é o que mantém sessões longas afiadas.

## Os dois comandos essenciais

Dois comandos cobrem quase toda situação. O truque é casar o comando com a sua intenção.

<Flashcards cards={[{front: "/clear", back: "Apaga a conversa e começa do zero. Use quando mudar para uma tarefa NÃO RELACIONADA. A maneira mais barata de recuperar um contexto limpo e focado."}, {front: "/compact", back: "Resume a conversa até o momento em uma forma compacta — mantém a essência enquanto libera espaço. Use quando quiser CONTINUAR a mesma tarefa mas a janela estiver ficando cheia."}]} />

Decida com uma pergunta: esta é uma tarefa nova ou a mesma?

```mermaid
flowchart TD
  A{New task or same task?} -->|New / unrelated| B[/clear]
  A -->|Same task, context full| C[/compact]
```

## Compactação automática

Quando o contexto se aproxima do limite, o Claude Code pode fazer **compactação automática** para que o trabalho continue sem uma parada brusca. É uma rede de segurança — mas usar `/clear` proativamente entre as tarefas dá resultados melhores do que depender dela.

<Callout type="tip" items={["Trate a compactação automática como um recurso de último caso, não como estratégia. Limpar entre tarefas não relacionadas mantém o contexto mais limpo do que deixar a janela encher até a compactação automática entrar em ação."]} />

Além dos dois comandos, quatro hábitos mantêm seu contexto enxuto desde o início.

:::tip Sintoma → solução
Se o Claude começar a esquecer uma instrução anterior ou a se repetir, a janela provavelmente está cheia. `/compact` (mesma tarefa) ou `/clear` (nova tarefa) costuma resolver na hora. Mais em [Solução de Problemas](/docs/contribute/troubleshooting).
:::
