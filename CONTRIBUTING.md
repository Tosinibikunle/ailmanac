# Contributing to AILmanac

Thank you — AILmanac is only *complete and always-fresh* because people like you
add to it. You do **not** need to be a developer. Fixing a typo, adding a
glossary term, sharing a template, or updating something Anthropic just shipped
are all genuinely valuable contributions.

There's a friendly, illustrated version of this guide on the site:
**docs/contribute/contribute-in-10-minutes**.

## Quick start (browser, no setup)

1. Find the page on the site and click **"Edit this page"** at the bottom.
2. Click the ✏️ pencil on GitHub (it forks the repo for you).
3. Edit the Markdown, then **propose changes** → **open a pull request**.
4. A maintainer reviews it. Done. 🎉

## Local setup

Requires Node.js (see `.nvmrc`).

```bash
npm install
npm run start     # live preview at http://localhost:3000
npm run build     # MUST pass — it fails on broken links
```

## The content contract

Every page follows these rules so quality stays high and review stays fast:

1. **Tag the level.** Start the body with `<LevelBadge level="beginner" />`
   (`beginner` | `intermediate` | `advanced` | `all`). These components are
   registered globally — do **not** import them.
2. **Cite sources for volatile facts.** Anything about model names, prices,
   limits, exact flags, or features that change must link the official source
   and carry a stamp:
   `<VerifyNote lastVerified="YYYY-MM-DD" source="https://docs.anthropic.com/...">…</VerifyNote>`.
   **Never hard-code model facts** (IDs/prices/context) in prose — link to
   `docs/whats-new/models-and-pricing` (rendered from `data/models.json`).
3. **Be opinionated, then honest.** Give the recommended way first, then note
   alternatives. Don't duplicate the official docs — link to them. They are the
   source of truth.
4. **Plain English, examples over theory.** Short sentences. Show, don't tell.
5. **Front-matter** on every doc:

   ```markdown
   ---
   sidebar_position: 2
   title: "Your Page Title"
   description: "One sentence for search and previews."
   ---

   <LevelBadge level="beginner" />

   Your first paragraph…
   ```

See `docs/contribute/style-guide` for the full style guide, and
`docs/contribute/fact-verification` for the freshness workflow.

## Adding a page

Create a `.mdx` file under the right `docs/<section>/` folder. The sidebar is
**autogenerated** from the folder structure — no manual link lists. Order with
`sidebar_position` and per-folder `_category_.json`.

## Translations (i18n)

English is the canonical source. To own a locale, see
`docs/contribute/translation-playbook`. We only enable a locale in production
once someone commits to maintaining it, to avoid stale partial translations.

## Pull request checklist

- [ ] `npm run build` passes locally (no broken links).
- [ ] Level badge present; volatile facts carry a `<VerifyNote>` with a source.
- [ ] Links use existing `/docs/...` paths or official external URLs.
- [ ] Commits are signed off (`git commit -s`) — see below.

## Developer Certificate of Origin (DCO)

We use the lightweight [DCO](https://developercertificate.org/) instead of a
CLA. Add a sign-off to each commit:

```bash
git commit -s -m "docs: add voice mode page"
```

This certifies you wrote the contribution (or have the right to submit it) and
agree to license your prose under **CC BY 4.0** and code under **MIT**.

## Be kind

We follow the Contributor Covenant — see `CODE_OF_CONDUCT.md`.
